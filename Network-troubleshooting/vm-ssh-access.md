# Настройка удалённого доступа и настройка статической сети в Linux
Первая моя проблема с которой столкнулся в самом начале своего развития!

## Проблема (Issue) 
При создании новой виртуалки отсутствовала возможность удалённого подключения через терминал (MobaXterm). Виртуальная машина находилась в изолированной сети (NAT) без проброса портов, а IP-адрес вообще не назначался

## Решение  
#### 1. Изменение сетевой топологии
Сменил режим работы сетевого адаптера в гипервизоре с NAT на Bridged Adapter. Это позволило виртуалке получить адрес в одной подсети с основным рабочим диапазоном 

#### 2. Настройка статического IP
Для того, чтобы адрес хотя бы появился и не менялся после перезагрузки системы, я выполнил настройку конфига netplan

<details><summary> Пример стандартного 50-cloud-init.yaml </summary>

```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s3:
      dhcp4: false
      addresses: [<IP>/24]
      routes:
        - to: default
          via: 192.168.0.1
      nameservers:
        addresses: [8.8.8.8, 1.1.1.1]
```
</details>

#### 3. Получение доступа
- Установил из запустил демон SSH. ```sudo apt install openssh-server && sudo systemctl enable --now ssh```
- Открыл порты фаейрволла (Ubuntu через ```ufw allow 22```, в CentOS через ```firewall-cmd --permanent --add-service=ssh``` )

## Что понял из этого
- Изучил разницу сетевых режимов, NAT и Bridge
- Впервые поработал с YAML-конфигурацией и осознал насколько это удобный формат
- Научился читать `ip a` и работать с `ss -tulpn `

## Почему чаще всего использую Bridge, а не NAT? 
- Если много виртуальных машин, то для каждой нужно использовать отдельный порт. Путаница и дополнительные порты занимает
- Если на виртуальной машине есть сложные сервисы вроде k8s, то придётся пробрасывать множество портов вручную
